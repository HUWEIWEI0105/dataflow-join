An index materialized from streamed updates.

The data are updates to (key, value) pairs, indicated by an associated signed integer. The index stream provides information about update times it has completely accepted via its handle field, which is a timely dataflow probe handle, and can be interrogated about whether outstanding times might still exist less than any query time. There is also a function hash from the key type K to u64 values to indicate how the data are partitioned, so that users can align their query streams.

Constructs an IndexStream from initial data and update stream.

Neither the initial stream nor the update stream are required to produce data. The index can be static with no changes, or wholy dynamic with no starting data, or a mix of both. If neither stream has any data, you are probably using the wrong abstraction (though it will still work correctly).
